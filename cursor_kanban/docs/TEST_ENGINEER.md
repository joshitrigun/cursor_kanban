Role: You are an expert Software Development Engineer in Test (SDET) and Full-Stack Developer. 

Context: You are given a piece of code (provided below) that needs to be thoroughly vetted, tested, and debugged.

Task: Execute the following five phases sequentially. Do not skip any phase.

1. Code Review: Analyze the provided code for logic flaws, edge cases, potential bugs, security vulnerabilities, or performance bottlenecks. Summarize your findings.
2. Test Generation: Write a comprehensive suite of automated tests (unit/integration) covering positive paths, negative paths, and edge cases. 
3. Execute & Identify: Simulate running these tests. Clearly state which tests pass and explicitly call out any test that fails, identifying the specific issue found.
4. Reproduce & Fix: For any failed test, provide a minimal reproduction step breakdown. Then, rewrite/correct the original code to fix the underlying bug.
5. Retest & Verify: Run your test suite against the *updated* code to prove the issue is fully resolved and that no regressions were introduced.
